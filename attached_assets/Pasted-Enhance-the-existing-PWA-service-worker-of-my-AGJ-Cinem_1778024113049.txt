Enhance the existing PWA service worker of my "AGJ Cinema" app to block popup ads and malicious redirects, specifically when streaming content through the embedded iframe player.

**Background:** The app uses an iframe to embed streams, and the external provider triggers popup windows and unwanted redirects. We need the service worker to intercept these.

**Your Task: Modify the service worker file (usually `sw.js` or `workbox-config.js`) to add an ad-blocking layer.**

1.  **Intercept all fetch requests** that happen within the app's scope.
2.  **Block known ad/tracking domains:** Add a list of common ad-network and popup domains to a blocklist. If a request URL matches any of these, block it immediately and return an empty response. The blocklist should include domains like `doubleclick.net`, `googleadservices.com`, `googlesyndication.com`, `popads.net`, `propellerads.com`, and any other major ad-serving domains you know.
3.  **Strip query parameters used for redirects:** Some popups work by adding specific parameters to URLs. Intercept requests and remove or neutralize query parameters like `&popup=true` or `&redirect_url=`.
4.  **Handle navigation requests:** If a navigation event (like a popup window opening) is detected and its target is not the main app or the allowed streaming domains (`vaplayer.ru`, `vidapi.ru`, `tmdb.org`, `image.tmdb.org`), block it and log the attempt to the console.
5.  **Ensure core functionality is not broken:**
    -   `vaplayer.ru` — allow all requests
    -   `vidapi.ru` — allow all requests
    -   `api.themoviedb.org` — allow all requests
    -   `image.tmdb.org` — allow all images
    -   `firebaseio.com` (if used for Auth) — allow all requests

**Verification:**
- The streaming iframe should still load and play videos normally.
- Popup windows should be blocked when clicking on the video area.
- The browser console should show logs like "Blocked ad request to: [domain]" when the service worker intercepts a blocked request.
- The main AGJ Cinema app must function flawlessly with all its existing features (Favorites, Auth, etc.).
